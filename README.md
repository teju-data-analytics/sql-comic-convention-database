# Comic Convention Database

Author: Maria Tejaswini Polishetty  
Project Type: Database Design & SQL Implementation  

## Project Overview

This project demonstrates the design and implementation of a relational database for managing a comic book convention. The database is designed to store and organize information about convention attendees, event schedules, artist booths, merchandise, and attendee itineraries.

The goal of the system is to efficiently track convention activities and manage relationships between different entities in the event.

## Database Tables

The database contains the following main tables:

• ConventionAttendee – stores attendee information  
• EventSchedule – stores convention events and schedules  
• ArtistBooths – stores artist and booth information  
• Merchandise – stores products available at the convention  
• AttendeeItinerary – tracks attendee participation in events and purchases

## Database Relationships

The tables are connected through relational keys to maintain structured data management.

Examples of relationships include:

• Attendees attending convention events  
• Artists participating in booths  
• Merchandise available for purchase  
• Attendee itinerary tracking events and products
